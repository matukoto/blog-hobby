## 現状分析  

### 既存構成  

- **プロジェクト**: SvelteKit (Svelte 5) + MDSvex + Cloudflare Workers  
- **既存コンテンツ**: `urara/` ディレクトリに15件以上の記事が存在  
  - 各記事は `urara/[slug]/+page.md` 形式（ディレクトリ構造）  
  - Frontmatter で `title`, `created`, `updated`, `tags`, `image` などを管理  
  - 画像は `urara/assets/` に配置、記事から `/assets/svelte.png` のように参照  
- **現在のトップページ**: "Welcome to SvelteKit" の初期状態  

### 既存ドキュメント  

`docs/UI.md` に以下の構想が記載されていた：  

```  
src/  
  ├── lib/  
  │    └── posts/ ( .md を入れる)  
  └── routes/  
       ├── +page.svelte (一覧を表示)  
       ├── [slug]/  
       │    └── +page.svelte (記事を表示)  
       └── api/  
            └── posts/  
                 └── +server.ts (全記事のメタデータを返す)  
```  

## 設計検討プロセス  

### 選択肢の整理  

#### 1. コンテンツの配置方法  

以下の3つの選択肢を提示：  

**選択肢1: urara/ をそのまま活用する**  

- メリット: 既存記事をそのまま使える、移行不要  
- デメリット: `urara` という名前が固定される  
**選択肢2: src/lib/posts/ に移行する** ✅ **採用**  
- メリット: 構造がシンプル、`docs/UI.md` の構想通り  
- デメリット: 移行作業が必要  

**選択肢3: ハイブリッド**  

- コンテンツは `urara/` に残し、API で読み込み  

→ **決定**: 選択肢2を採用  

#### 2. ファイル構造  

- **A案**: `src/lib/posts/first.md`（ファイル単体、slug はファイル名から） ✅ **採用**  
- **B案**: `src/lib/posts/first/index.md`（現在の構造を維持）  
- **C案**: フラットに `src/lib/posts/` に全部置く  

→ **決定**: A案を採用（シンプルで扱いやすい）  

#### 3. 画像ファイルの扱い  

- **パターン1**: `static/assets/` に移動（公開ディレクトリ） ✅ **採用**  
- **パターン2**: `src/lib/assets/` に移動して Vite でインポート  
- **パターン3**: 既に `static/` にコピー済みなら移行不要  

→ **決定**: パターン1を採用（Frontmatter の `image: '/assets/svelte.png'` がそのまま使える）  

#### 4. ルーティング設計  

- URL 形式: `https://example.com/article/first` （`/article` プレフィックス付き）  
- 構造:  

```  
src/routes/  
  ├── +page.svelte              # トップページ（記事一覧ではない）  
  ├── article/  
  │   └── [slug]/  
  │       ├── +page.svelte      # 記事詳細  
  │       └── +page.ts          # データ読み込み  
  └── api/posts/  
      └── +server.ts            # 全記事メタデータAPI  
```  

#### 5. 実装の優先順位  

- **優先度1**: 記事詳細ページ（`article/[slug]/+page.svelte`）✅ **最初に実装**  
- 優先度2: APIエンドポイント（`api/posts/+server.ts`）  
- 優先度3: 記事一覧ページ（`+page.svelte`）  

→ **決定**: 1 → 2 → 3 の順番で実装  

## 確定した実装計画

#### ステップ1-3: 記事読み込みロジック（`src/lib/posts.ts`）  

実装する機能：  

- `import.meta.glob()` で全 Markdown 読み込み  
- Frontmatter のパース  
- slug から記事を取得する関数  
- 型定義（`Post` interface）  

```typescript  
// 期待される型定義  
export interface Post {  
  slug: string;  
  title: string;  
  created: string;  
  updated?: string;  
  published?: string;  
  tags?: string[];  
  image?: string;  
  content: string; // HTML化されたコンテンツ  
}  

export async function getPost(slug: string): Promise<Post | null>  
export async function getAllPosts(): Promise<Post[]>  
```  

#### ステップ1-4: ルーティング実装  

**`src/routes/article/[slug]/+page.ts`**  

- `load` 関数で `getPost(params.slug)` を呼び出し  
- 記事が見つからない場合は404エラー  

**`src/routes/article/[slug]/+page.svelte`**  

- 記事のタイトル、メタデータ、本文を表示  
- シンプルなスタイリング
