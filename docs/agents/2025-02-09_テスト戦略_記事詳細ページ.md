## 1. Unit テスト例

### 対象: `src/lib/posts.ts`（記事読み込みロジック）

#### テストファイル: `src/lib/posts.spec.ts`

```typescript
import { describe, it, expect } from 'vitest';
import { getPost, getAllPosts, type Post } from './posts';

describe('posts module', () => {
  describe('getPost', () => {
    it('should return post when slug exists', async () => {
      const post = await getPost('first');
      
      expect(post).toBeDefined();
      expect(post?.slug).toBe('first');
      expect(post?.title).toBe('SvelteKit でブログを作ってみた');
      expect(post?.content).toContain('感想');
    });

    it('should return null when slug does not exist', async () => {
      const post = await getPost('non-existent-slug');
      
      expect(post).toBeNull();
    });

    it('should parse frontmatter correctly', async () => {
      const post = await getPost('first');
      
      expect(post?.created).toBe('2024-09-20');
      expect(post?.tags).toEqual(['svelte', 'cloudflare', 'tech']);
      expect(post?.image).toBe('/assets/svelte.png');
    });

    it('should convert markdown to HTML', async () => {
      const post = await getPost('first');
      
      // Markdown の見出しが HTML に変換されている
      expect(post?.content).toContain('<h2');
      expect(post?.content).toContain('</h2>');
    });
  });

  describe('getAllPosts', () => {
    it('should return all posts', async () => {
      const posts = await getAllPosts();
      
      expect(posts.length).toBeGreaterThan(0);
      expect(posts.every(post => post.slug)).toBe(true);
      expect(posts.every(post => post.title)).toBe(true);
    });

    it('should sort posts by created date descending', async () => {
      const posts = await getAllPosts();
      
      for (let i = 1; i < posts.length; i++) {
        const prevDate = new Date(posts[i - 1].created);
        const currDate = new Date(posts[i].created);
        expect(prevDate >= currDate).toBe(true);
      }
    });

    it('should filter out posts without published date if specified', async () => {
      const posts = await getAllPosts({ onlyPublished: true });
      
      expect(posts.every(post => post.published)).toBe(true);
    });
  });
});
```

#### なぜ Unit テストを使うか？

- **データ処理ロジックのテスト**: Markdown のパース、Frontmatter の抽出など
- **境界値のテスト**: 存在しない slug、不正なデータなど
- **高速**: ブラウザ起動不要で即座に実行できる
- **ロジックの正確性**: ビジネスルールが正しく動くか確認

## 2. Component テスト例

### 対象: `src/routes/article/[slug]/+page.svelte`（表示コンポーネント）

#### テストファイル: `src/routes/article/[slug]/page.svelte.spec.ts`

```typescript
import { page } from 'vitest/browser';
import { describe, expect, it } from 'vitest';
import { render } from 'vitest-browser-svelte';
import ArticlePage from './+page.svelte';

describe('/article/[slug]/+page.svelte', () => {
  const mockPost = {
    slug: 'test-post',
    title: 'テスト記事',
    created: '2024-01-01',
    updated: '2024-01-02',
    tags: ['svelte', 'test'],
    image: '/assets/test.png',
    content: '<h2>見出し</h2><p>本文テキスト</p>'
  };

  describe('記事の基本表示', () => {
    it('should render article title', async () => {
      render(ArticlePage, { data: { post: mockPost } });
      
      const title = page.getByRole('heading', { level: 1 });
      await expect.element(title).toHaveText('テスト記事');
    });

    it('should render article content', async () => {
      render(ArticlePage, { data: { post: mockPost } });
      
      const heading = page.getByRole('heading', { level: 2 });
      await expect.element(heading).toHaveText('見出し');
      await expect.element(page.getByText('本文テキスト')).toBeInTheDocument();
    });

    it('should render created date', async () => {
      render(ArticlePage, { data: { post: mockPost } });
      
      await expect.element(page.getByText(/2024-01-01/)).toBeInTheDocument();
    });

    it('should render article image when provided', async () => {
      render(ArticlePage, { data: { post: mockPost } });
      
      const img = page.getByRole('img');
      await expect.element(img).toHaveAttribute('src', '/assets/test.png');
    });

    it('should not render image when not provided', async () => {
      const postWithoutImage = { ...mockPost, image: undefined };
      render(ArticlePage, { data: { post: postWithoutImage } });
      
      const images = await page.getByRole('img').all();
      expect(images.length).toBe(0);
    });
  });

  describe('タグ表示', () => {
    it('should render all tags', async () => {
      render(ArticlePage, { data: { post: mockPost } });
      
      await expect.element(page.getByText('svelte')).toBeInTheDocument();
      await expect.element(page.getByText('test')).toBeInTheDocument();
    });

    it('should render tags as links', async () => {
      render(ArticlePage, { data: { post: mockPost } });
      
      const svelteLink = page.getByRole('link', { name: 'svelte' });
      await expect.element(svelteLink).toHaveAttribute('href', '/tags/svelte');
    });

    it('should not render tag section when no tags', async () => {
      const postWithoutTags = { ...mockPost, tags: [] };
      render(ArticlePage, { data: { post: postWithoutTags } });
      
      await expect.element(page.getByText('Tags:')).not.toBeInTheDocument();
    });
  });

  describe('メタデータ表示', () => {
    it('should show updated date when different from created', async () => {
      render(ArticlePage, { data: { post: mockPost } });
      
      await expect.element(page.getByText(/更新/)).toBeInTheDocument();
      await expect.element(page.getByText(/2024-01-02/)).toBeInTheDocument();
    });

    it('should not show updated date when same as created', async () => {
      const postSameDate = { ...mockPost, updated: '2024-01-01' };
      render(ArticlePage, { data: { post: postSameDate } });
      
      await expect.element(page.getByText(/更新/)).not.toBeInTheDocument();
    });
  });
});
```

#### なぜ Component テストを使うか？

- **UI の描画確認**: 正しく要素が表示されるか
- **条件分岐のテスト**: 画像がある/ない、タグがある/ない など
- **インタラクションのテスト**: クリック、ホバーなどのユーザー操作（今回は少ない）
- **アクセシビリティ確認**: role、aria属性など
- **スタイルの確認**: 要素の表示/非表示、クラス名など

## 3. E2E テスト例

### 対象: 記事詳細ページ全体のフロー

#### テストファイル: `e2e/article.test.ts`

```typescript
import { expect, test } from '@playwright/test';

test.describe('記事詳細ページ', () => {
  test('記事が正しく表示される', async ({ page }) => {
    // 実際の記事ページにアクセス
    await page.goto('/article/first');

    // タイトルが表示される
    await expect(page.locator('h1')).toContainText('SvelteKit でブログを作ってみた');

    // 記事本文が表示される
    await expect(page.locator('h2').first()).toContainText('感想');

    // メタデータが表示される
    await expect(page.getByText(/2024-09-20/)).toBeVisible();

    // タグが表示される
    await expect(page.getByText('svelte')).toBeVisible();
    await expect(page.getByText('cloudflare')).toBeVisible();
  });

  test('存在しない記事は404エラー', async ({ page }) => {
    const response = await page.goto('/article/non-existent');
    
    expect(response?.status()).toBe(404);
    await expect(page.locator('body')).toContainText('404');
  });

  test('記事からタグページへ遷移できる', async ({ page }) => {
    await page.goto('/article/first');

    // タグをクリック
    await page.getByRole('link', { name: 'svelte' }).click();

    // タグページに遷移
    await expect(page).toHaveURL('/tags/svelte');
    await expect(page.locator('h1')).toContainText('svelte');
  });

  test('記事一覧から記事詳細へ遷移できる', async ({ page }) => {
    // トップページ（記事一覧）にアクセス
    await page.goto('/');

    // 記事をクリック
    await page.getByRole('link', { name: /SvelteKit でブログを作ってみた/ }).click();

    // 記事詳細ページに遷移
    await expect(page).toHaveURL('/article/first');
    await expect(page.locator('h1')).toContainText('SvelteKit でブログを作ってみた');
  });

  test('画像が正しく読み込まれる', async ({ page }) => {
    await page.goto('/article/first');

    // 画像要素が存在する
    const img = page.locator('img').first();
    await expect(img).toBeVisible();

    // 画像が実際に読み込まれている（broken image ではない）
    const naturalWidth = await img.evaluate((el: HTMLImageElement) => el.naturalWidth);
    expect(naturalWidth).toBeGreaterThan(0);
  });

  test('モバイル表示でも正しく表示される', async ({ page }) => {
    // モバイルビューポート
    await page.setViewportSize({ width: 375, height: 667 });
    await page.goto('/article/first');

    await expect(page.locator('h1')).toBeVisible();
    await expect(page.locator('h2').first()).toBeVisible();

    // スクロールしても表示が崩れない
    await page.evaluate(() => window.scrollTo(0, document.body.scrollHeight));
    await expect(page.locator('h1')).toBeVisible();
  });
});
```

#### なぜ E2E テストを使うか？

- **実際のユーザー体験のテスト**: ブラウザでの実際の動作確認
- **ページ間の遷移**: ルーティング、ナビゲーションの確認
- **リソースの読み込み**: 画像、CSS、JS が正しく読み込まれるか
- **レスポンシブ対応**: モバイル、タブレット、デスクトップでの表示確認
- **統合テスト**: フロントエンド + バックエンド全体の動作確認

## テストの使い分けまとめ

| テスト種別 | 対象 | 目的 | 実行速度 |
|-----------|------|------|---------|
| **Unit** | `src/lib/posts.ts` | データ処理ロジックの正確性 | ⚡ 超高速 |
| **Component** | `+page.svelte` | UI の描画・条件分岐 | 🚀 高速 |
| **E2E** | ページ全体 | ユーザー体験・統合動作 | 🐢 低速 |

### TDD での実装順序（記事詳細ページの場合）

#### Phase 1: Unit テスト（Red）
```bash
# 1. posts.spec.ts を書く（失敗するテスト）
pnpm test:unit src/lib/posts.spec.ts
```

#### Phase 2: Unit テスト（Green）
```bash
# 2. posts.ts を実装（テストが通る）
pnpm test:unit src/lib/posts.spec.ts
```

#### Phase 3: Unit テスト（Refactor）
```bash
# 3. リファクタリング（テストが通ることを確認）
pnpm test:unit src/lib/posts.spec.ts
```

#### Phase 4: Component テスト（Red → Green → Refactor）
```bash
# 4. page.svelte.spec.ts を書く → +page.svelte を実装
pnpm test:unit src/routes/article/[slug]/page.svelte.spec.ts
```

#### Phase 5: E2E テスト（Red → Green → Refactor）
```bash
# 5. article.test.ts を書く → 統合的に動作確認
pnpm test:e2e
```

### 優先順位

1. **Unit テスト**: ロジックが複雑な場合は必須。最初に書く
2. **Component テスト**: UI の条件分岐が多い場合に有効
3. **E2E テスト**: 主要なユーザーフローだけ書く（全パターンは不要）

### 書かなくて良いテスト

- **過度な Component テスト**: 「h1 が存在する」だけのテストは不要（E2E で十分）
- **フレームワークのテスト**: SvelteKit のルーティングそのもののテスト
- **サードパーティのテスト**: MDSvex のパース機能自体のテスト
